Spring Boot Hello World Example with Maven

This simple Hello World application demonstrates how to set up a Spring Boot project using Maven. We will create a basic REST API that returns a "Hello, World!" message when accessed via HTTP.

Steps to Create a Spring Boot Hello World Application with Maven

  1. Setup Maven for your project (in the pom.xml file).
  2. Create a Spring Boot application class with @SpringBootApplication.
  3. Define a REST controller that handles HTTP requests.
  4. Run the application using Maven command.

1. pom.xml Configuration

In your Maven project's pom.xml, you need to define the Spring Boot dependencies. Here’s the basic configuration:


      <project xmlns="http://maven.apache.org/POM/4.0.0"
               xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
               xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
          <modelVersion>4.0.0</modelVersion>
      
          <groupId>com.example</groupId>
          <artifactId>hello-world</artifactId>
          <version>1.0-SNAPSHOT</version>
          <packaging>jar</packaging>
      
          <dependencies>
              <dependency>
                  <groupId>org.springframework.boot</groupId>
                  <artifactId>spring-boot-starter-web</artifactId>
              </dependency>
              <dependency>
                  <groupId>org.springframework.boot</groupId>
                  <artifactId>spring-boot-starter-test</artifactId>
                  <scope>test</scope>
              </dependency>
          </dependencies>
      
          <build>
              <plugins>
                  <plugin>
                      <groupId>org.springframework.boot</groupId>
                      <artifactId>spring-boot-maven-plugin</artifactId>
                  </plugin>
              </plugins>
          </build>
      </project>

The spring-boot-starter-web dependency includes everything needed to create a web application, while spring-boot-starter-test includes libraries for testing.

2. Creating the Spring Boot Application Class

Next, create the main application class annotated with @SpringBootApplication, which is the entry point of the application.


      import org.springframework.boot.SpringApplication;
      import org.springframework.boot.autoconfigure.SpringBootApplication;
      
      @SpringBootApplication
      public class HelloWorldApplication {
      
          public static void main(String[] args) {
              SpringApplication.run(HelloWorldApplication.class, args);
          }
      }

Here, the @SpringBootApplication annotation tells Spring Boot to automatically configure your application based on the dependencies and to scan for components in the same package.

3. Creating the REST Controller

Create a controller class to handle HTTP requests. In this case, it will return a simple "Hello, World!" message:


      import org.springframework.web.bind.annotation.GetMapping;
      import org.springframework.web.bind.annotation.RestController;
      
      @RestController
      public class HelloWorldController {
      
          @GetMapping("/hello")
          public String sayHello() {
              return "Hello, World!";
          }
      }

The @RestController annotation defines this class as a RESTful controller, and the @GetMapping("/hello") annotation maps the HTTP GET request to the sayHello() method.

4. Running the Application

Once the application is set up, you can run it using Maven. Open the terminal or command prompt and navigate to your project directory. Then, execute the following command:


      mvn spring-boot:run

This command will build and start your Spring Boot application. By default, the application will run on port 8080.

5. Accessing the Application

After running the application, you can open your web browser and go to the following URL:

http://localhost:8080/hello

You should see the message: "Hello, World!"

Core Concepts of Spring Boot

Spring Boot is a framework that simplifies Java development by providing default configurations and essential tools. In this section, we'll cover key concepts of Spring Boot, such as its application structure, embedded web servers, and Spring Boot starters.

1. Spring Boot Application Structure

The basic structure of a Spring Boot application is simple and follows a convention over configuration philosophy. Here is a typical structure:


/src
    /main
        /java
            /com
                /example
                    /demo
                        Application.java   
        /resources
            application.properties   
            /static  
            /templates  
    /test
        /java
            /com
                /example
                    /demo
                        ApplicationTests.java

2. Anatomy of a Spring Boot Application

A Spring Boot application typically includes the following components:

  • Application Class: The class annotated with @SpringBootApplication that serves as the entry point for the application.
  • Configuration File: application.properties or application.yml for externalized configuration.
  • Controller Class: Handles HTTP requests and returns responses, typically annotated with @RestController.
  • Service Class: Contains business logic and interacts with repositories or databases.
  • Repository Class: Manages data access, typically using Spring Data JPA or similar frameworks.

3. Main Application Class with @SpringBootApplication Annotation

The @SpringBootApplication annotation is a combination of three important annotations:

  • @Configuration: Indicates that this class contains configuration settings.
  • @EnableAutoConfiguration: Tells Spring Boot to automatically configure the application based on the dependencies in the classpath.
  • @ComponentScan: Enables component scanning so that Spring Boot can find and register beans.

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);  // Launch the application
    }
}

This Application class is the entry point for the Spring Boot application. The SpringApplication.run() method starts the embedded web server and loads the Spring application context.

4. Embedded Web Server (Tomcat/Jetty)

Spring Boot comes with embedded web servers like Tomcat, Jetty, and Undertow, so you don't need to install a separate web server. By default, Spring Boot uses Tomcat as the embedded web server.

Spring Boot automatically configures the embedded web server based on the application’s dependencies.

Example of Embedded Web Server:

When you create a Spring Boot application, an embedded Tomcat server is automatically configured. You can override this configuration by adding another embedded server, such as Jetty:


dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-jetty'  // Use Jetty instead of Tomcat
}

5. Working with Embedded Servers

Spring Boot automatically starts the embedded server when the application is launched. For example, using Tomcat, the default port for the application is 8080.

If you want to change the port or other server settings, you can do so in the application.properties file:


server.port=9090

When you run the application, the embedded server listens on the specified port and serves the application.

6. Spring Boot Starters

Spring Boot Starters are pre-configured templates or dependencies that simplify adding specific functionalities to your Spring Boot applications. These starters bundle together the dependencies required for specific tasks such as building web applications or working with databases.

What are Starters?

Starters are a set of dependencies that you can add to your project for specific functionality. For example:

  • spring-boot-starter-web: For building web applications (RESTful services).
  • spring-boot-starter-data-jpa: For working with relational databases using Spring Data JPA.
  • spring-boot-starter-security: For adding security to your application.

Commonly Used Starters:

  • spring-boot-starter-web: Provides everything needed to build a web application, including Tomcat and Spring MVC.
  • spring-boot-starter-data-jpa: Enables integration with relational databases and ORM frameworks like Hibernate.
  • spring-boot-starter-security: Provides default security configurations for applications, including authentication and authorization.
  • spring-boot-starter-thymeleaf: For integrating Thymeleaf as the template engine for web applications.

Example: Adding Starters in pom.xml


      <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
      </dependency>

    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-data-jpa</artifactId>
    </dependency>

These starters allow you to quickly get up and running with web development, database interaction, and more.
